plan-orchestrator · public v1 runtime

Plan Orchestrator Workflow

A resumable, multi-stage pipeline that reads one approved markdown_playbook_v1, normalizes it into a runtime manifest, and executes items through isolated git worktrees with verification, dual audit, deterministic findings merge, and bounded repair loops.

ST05 Playbook Parse & Normalize orchestrator ▾

The approved markdown_playbook_v1 file is parsed into numbered sections and an ordered execution table. The public MarkdownPlaybookAdapter transforms raw rows into NormalizedPlan objects with explicit authored-vs-derived boundaries.

Adapter responsibilities:

  • Parse prerequisites (e.g., "1-5" → [01,02,03,04,05])
  • Derive change_profile (behavioral_code / docs_only / mixed)
  • Keep allowed_write_roots and requires_red_green as authored playbook fields
  • Keep execution_mode=codex and host_commands=[] as derived public-v1 runtime fields
  • Attach support sections by phase slug
  • Map manual gates and external checks per item
  • Normalize verification hints and repo-relative paths from the markdown contract
OutputNormalizedPlan JSON (schema-validated)

RESOLVE Resolve Requested Items orchestrator ▾

Resolves which items to process based on CLI input: --item (single), --items (comma-separated), or --next (first unfinished with prerequisites met).

Prerequisite checks: Each item's prerequisite_item_ids must all have terminal_state == "passed" before it can run. The orchestrator refuses to start items with unmet prerequisites.

Auto-advance: With --auto-advance, after an item passes, the orchestrator automatically selects the next eligible item up to --max-items.

Section 02

Per-Item Processing Loop

Each item is processed in an isolated git worktree through mutation → verification → dual audit → triage. Failed triage routes to fix/remediation rounds (looping back), or to terminal states.

ST10 Item Selected orchestrator ▾

Validates prerequisites are satisfied. If the item requires external evidence and none is provided, immediately transitions to ST120 BLOCKED_EXTERNAL. Otherwise, proceeds to worktree preparation.

Guardprerequisites_satisfied()
Guardrepo input availability check
ST15 Worktree Prepared git ▾

Creates an isolated git worktree branched from the run branch. Each item+attempt gets its own branch: orchestrator/item/{run_id}/{item_id}/attempt-{n}.

The worktree contains a .local/plan_orchestrator/packet/ directory where non-tracked artifacts (audit reports, playbook snapshots) are materialized as SHA256-pinned copies visible to the mutation agent.

Fix / Remediation Loop
ST30 Mutation Stage (Execute / Fix / Remediation) codex write sandbox ▾

Invokes Codex in workspace-write sandbox mode with the rendered prompt and output schema. The agent reads the item context packet, performs the required work (code changes, doc creation), and produces a structured report.

Three mutation variants:

  • execute — initial implementation (prompt: execution_codex.md)
  • fix — address actionable findings from triage (prompt: fix_codex.md)
  • remediation — deeper remediation after fix budget exhausted (prompt: remediation_codex.md)

After Codex finishes, the orchestrator runs a scope check (no forbidden paths, no out-of-scope changes), stages allowed changes, and creates a checkpoint commit.

Default timeout2h execute, 1.5h fix/remediation
Sandboxcodex exec -s workspace-write
Outputexecution_report.schema.json

ST40 Verification Gate orchestrator deterministic ▾

Runs the verification plan defined in the item context:

  1. Command groups — shell commands (e.g., forge test) with required/optional classification
  2. Artifact checks — file existence and content substring checks for deliverables
  3. Scope check — verifies committed changes stay within allowed_write_roots

Also asserts checkpoint authority: HEAD must not have drifted from the checkpoint commit, and the worktree must be clean.

Default timeout1h per command
Outputverification_report.schema.json
Moduleverification.py → git_checkpoint.py
ST50 Audit Packet Assembly orchestrator ▾

Assembles the audit packet: a SHA256-pinned manifest of all artifacts needed for review (candidate patch, verification report, mutation report, playbook snapshot, normalized plan). Artifacts are materialized as packet copies inside the worktree.

Generates separate prompts for the Codex and Claude auditors from templates, referencing the audit packet manifest paths.

Artifactcandidate.{cycle}.patch
Promptsaudit_codex.md, audit_claude.md
ST60→61 Dual-Model Audit codex claude read-only ▾

Two independent auditors review the frozen checkpoint — neither can modify the worktree:

  • ST60 — Codex Audit: codex exec in read-only sandbox. Reviews patch, verification results, and playbook compliance.
  • ST61 — Claude Audit: claude -p in plan permission mode with Read/Glob/Grep tools only. Independent review with alternate report normalization.

Both produce structured findings with severity, category, confidence, evidence, and recommended_action.

Outputaudit_report.schema.json

MERGE Deterministic Findings Merge orchestrator deterministic ▾

Merges findings from four sources: Codex audit, Claude audit, verification failures, and mutation report signals. Each finding receives a deterministic canonical_id via SHA256 hashing of (title, category, file_paths, evidence).

Duplicate findings across sources are merged by canonical_id (source_refs are combined, is_blocking is OR'd). Mutation report signals (blocked_external, needs_human_input, escalate, unresolved_items) generate synthetic findings.

Dedupcanonical_id = SHA256(title+category+paths+evidence)[:16]
Outputmerged_findings_packet.json

ST70 Triage codex read-only ▾

A Codex agent in read-only mode reviews all merged findings and produces an overall_decision. The orchestrator then preserves mutation-stage control signals — if the mutation report requested blocked_external or escalation, the triage decision is overridden to match.

Additionally, blocking repo-content findings from audit/verification that triage tried to pass are forcibly preserved as fix_required.

Section 04

Component Architecture

All modules live under automation/plan_orchestrator/. Each has a single responsibility.

runtime.py
PlanOrchestrator class · lifecycle engine
Central orchestration engine. Manages the full item lifecycle: worktree prep → mutation → verify → audit → triage → terminal routing. Contains the main processing loop and all stage-specific logic.
Core Engine
cli.py
build_parser() + main()
CLI entry point with 6 subcommands: list-items, show-item, run, resume, refresh-run, and mark-manual-gate. Thin wrapper that delegates to PlanOrchestrator methods.
IO / Interface
models.py
@dataclass models
Typed dataclasses for all domain objects: PlanItem, NormalizedPlan, SupportSection, RunState, ItemRunState, RuntimeOptions, WorktreeMetadata, ItemContext, RunEvent, LatestPaths.
Core Engine
state_machine.py
StateId enum + TRANSITIONS dict
Defines the 17-state FSM with allowed transitions. Provides assert_transition(), first_unfinished_item(), and prerequisites_satisfied() queries.
Core Engine
state_store.py
CRUD for RunState
Creates, loads, saves, and updates run state JSON files. All writes are schema-validated and atomic (write to temp, then rename). Appends timestamped events to the event log.
Infrastructure
playbook_parser.py
Markdown → structured data
Parses approved playbook Markdown into structured sections, subsections, and the ordered execution table. Extracts repo paths, code blocks, and numbered headings.
IO / Interface
adapters/markdown_playbook.py
MarkdownPlaybookAdapter
Public v1 adapter that transforms one approved markdown_playbook_v1 file into NormalizedPlan items. Enforces the authored-vs-derived contract, maps manual gates and external checks, and projects support sections into item context.
Adapter
worktree_manager.py
WorktreeManager class
Git operations: creates run branches, prepares isolated worktrees per item+attempt, fast-forwards run branch on pass. Enforces clean checkout and git identity.
Infrastructure
subprocess_runner.py
run_codex_stage() + run_claude_audit()
Subprocess wrappers for Codex and Claude CLI invocations. Handles output schema validation, JSON extraction from Claude responses, alternate report normalization, and "max effort" fallback retry.
Infrastructure
verification.py
run_verification()
Deterministic verification gate: executes shell commands, checks artifact existence/content, validates scope of committed changes. Produces a structured pass/fail/partial report.
Safety / Guardrail
git_checkpoint.py
Scope enforcement + checkpoints
Stages allowed changes (rejecting forbidden/out-of-scope paths), creates checkpoint commits, verifies post-checkpoint authority (no HEAD drift, no dirty files), and generates patches.
Safety / Guardrail
findings.py
Deterministic merge + dedup
Merges findings from 4 sources (Codex audit, Claude audit, verification, mutation signals) using canonical_id deduplication. Generates synthetic findings from mutation report control signals.
Core Engine
reporting.py
Artifact management
Builds artifact manifests with SHA256-pinned packet copies, renders prompt templates, writes manual gate records, escalation manifests, and pass summaries.
IO / Interface
validators.py
Utility functions
JSON schema validation, atomic file writes, SHA256 computation, path scope checking, slug generation, timestamp formatting, and path resolution utilities.
Infrastructure
config.py
Defaults + environment overrides
Configurable defaults for models (gpt-5.4, opus), timeouts (2h execute, 1h verify/audit, 30m triage, 1.5h fix/remediation), fix budgets (2 fix + 1 remediation round), paths, and environment variable overrides.

Section 05

CLI Reference

All commands are run via python automation/run_plan_orchestrator.py <command>.

list-items
Display all normalized plan items with phase, change profile, R/G requirement, and gate status.
--playbook <path> --format table|json
show-item
Show full details for a single normalized plan item.
--playbook <path> --item <id> --format text|json
run
Start a new orchestration run. Specify a single item, a comma-separated list, or --next for the first eligible item.
--playbook <path> --item <id> | --items <id,id> | --next --auto-advance --max-items <n> --external-evidence-dir <path>
resume
Resume an existing run from its last known state. Resets blocked_external or escalated items for a fresh attempt.
--run-id <id> --external-evidence-dir <path> --auto-advance
refresh-run
Retarget a saved run onto a descendant branch and rebuild its normalized plan from the saved playbook snapshot.
--run-id <id> --retarget-run-branch-to <ref>
mark-manual-gate
Record a human decision on a pending manual gate. Approved gates fast-forward the run branch; rejected gates escalate.
--run-id <id> --item <id> --decision approved|rejected --by <name> --note <text> --evidence-path <path>
